# 基本操作

## 列出全部板卡列表
```bash
marstool chaif
```

## 列出指定刀片的板卡列表
```bash
# N代表刀片号
marstool nodif N
```

## 开启板卡ADB模式
```bash
# 代表N号刀片上的M号板卡
marstool socusbin N M 

# 确认开启ADB模式的板卡ID
adb devices

adb -s ID shell

# 若只有一张板卡开启ADB模式，则可不指定ID连接
adb shell
```

## 重启板卡
```bash
marstool socpwr N M rst
```

## 配置板卡IP
```bash
# IP是该板卡对应刀片的第一章板卡的IP，该命令会一次性顺序配置整个刀片所有的板卡IP
marstool nodif N ip 192.168.1.100 mask 255.255.255.0 gw 192.168.0.1 dns 8.8.8.8

# 配置IP后需要重启板卡的网络模块，可以选择ADB连接后重启networking，也可以直接重启板卡
marstool socpwr N M rst
```

## 开启板卡刷机模式
```bash
# 开启了刷机模式的板卡会在IP后有一个*
marstool socqfil N M on

# 查看开启了刷机模式的板卡locationid
upgrade_tool ld
```

# 板卡刷机

## 开启刷机模式
```bash
marstool socqfil N M on

upgrade_tool ld

upgrade_tool uf /userdata/roms/factory/baserom/update.img
```

# 板卡更新内核

## 查看内核版本
```bash
cat /prod/cpuinfo
```

## 开启刷机模式
```bash
marstool socqfil N M on
```

## 更新内核
```bash
upgrade_tool di -boot /userdata/boot-x.x.x.img
```

# 串口连接板卡排查故障
```bash
# 先关闭后台服务对BMC串口的占用
systemctl stop marssoc-console

# 执行命令将指定SoC串口连接到BMC
marstool socuartin N M on

# 使用串口程序minicom查看soc串口输出，
# 如果有输出，则可以看到输出内容，如果没有输出内容则soc可能已经挂死，需要重启
minicom -D /dev/ttyS0 -b 1500000

ctrl+A 然后按一下o，第三选项，倒数第二个是调整为no，然后新开一个终端重启这张卡，如果仍然不能输入，则标记异常：进不了系统

marstool socpwr N M rst

# 退出串口的方法1
ctrl+A 然后按一下Q
```
